# Инструкция по обработке запросов об изменениях (Release Changes)

При поступлении вопроса вида "Какие изменения были в витрине {datamart_name} за последний год?", необходимо следовать данному алгоритму:

### 1. Навигация и поиск источника
*   На основной странице витрины (там, где расположена ссылка на S2T) необходимо найти ссылку с текстом **"Изменения в релизах"**.
*   Перейти по найденной ссылке на дочернюю или связанную страницу.

### 2. Парсинг страницы изменений
На странице "Изменения в релизах" необходимо извлечь следующие данные:
*   **Версия/Дата:** Заголовки разделов (например, "Версия 20251215").
*   **Ярлыки (Tags):** Значения из макросов `status-macro` (например, "ИСПРАВЛЕНИЕ", "НОВОЕ", "ИЗМЕНЕНИЕ").
*   **Суть изменений:** Текстовое описание, следующее за ярлыком (например, "Корректировка атрибутивного состава...").
*   **Ссылки на задачи:** Найти ссылки на задачи в Jira (например, `SSDWH-2813`), которые обычно вложены в блоки `jira-issue`.

### 3. Обогащение данными из Jira (Deep Parsing)
Для каждой найденной задачи необходимо:
1.  Перейти по ссылке в Jira.
2.  Спарсить поле **"Создано"** (Created) — зафиксировать дату начала работ.
3.  В разделе **"Активность" (Activity)** -> вкладка **"Все" (All)** найти последнее (самое свежее) поле, название которого совпадает с ярлыком на странице Confluence (например, "ИСПРАВЛЕНИЕ").
4.  Извлечь значение этого поля.

### 4. Формирование ответа
Итоговый ответ должен быть структурирован в хронологическом порядке (от новых к старым) и включать:
*   Дату релиза (из Confluence).
*   Тип изменения (Ярлык).
*   Краткое описание сути.
*   Дату создания задачи (из Jira).
*   Статус/результат из последней активности Jira.
*   Ссылку на первоисточники (Confluence и Jira).
