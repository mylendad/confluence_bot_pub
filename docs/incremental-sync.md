# Инкрементальное обновление (Incremental Sync)

Этот документ описывает механизм оптимизированного обновления данных в системе, который позволяет избегать лишней нагрузки на сеть и вычислительные ресурсы.

## Общий принцип: Metadata-First

Система не скачивает и не парсит файлы S2T при каждом запуске. Вместо этого она следует стратегии проверки «от легкого к тяжелому».

### Схема процесса

1.  **Сбор метаданных (Collect):**
    *   Бот запрашивает через API Confluence только метаданные файлов (ID версии, размер, дата обновления).
    *   Формируется текстовый снимок метаданных (Metadata Snapshot).
2.  **Сравнение хэшей (Compare):**
    *   Считается `sha256` от снимка метаданных.
    *   Результат сравнивается с тем, что хранится в таблице `s2t_state`.
    *   Если хэш совпал — переход к следующей витрине (данные не изменились).
3.  **Загрузка (Download):**
    *   Если метаданные изменились, файл скачивается в папку `data/raw`.
4.  **Проверка содержимого (Verify):**
    *   Считается `sha256` от самого тела файла (`content_hash`).
    *   Если он совпал с предыдущим (например, в Confluence просто обновили версию вложения без изменения файла) — обновление останавливается.
5.  **Парсинг и Индексация (Parse & Index):**
    *   Только если изменился `content_hash`, запускается тяжелый процесс парсинга Excel/CSV и пересборка фрагментов для этой конкретной витрины.

## Преимущества

*   **Экономия трафика:** Мы не скачиваем десятки мегабайт Excel-файлов, если они не менялись.
*   **Скорость:** Проверка метаданных занимает секунды, в то время как полный парсинг и индексация могут длиться минуты.
*   **Точечные обновления:** При изменении одной витрины пересобирается только её часть индекса. Остальные данные остаются нетронутыми.

## Логирование изменений (Change Log)

В процессе инкрементального обновления работает `DiffService`. Он сравнивает старый список атрибутов витрины с новым и фиксирует:
*   `added`: новые поля.
*   `removed`: удаленные поля.
*   `modified`: изменения в логике или типах данных.

Эти данные попадают в таблицу `change_log`, что позволяет боту отвечать на вопросы об истории изменений («Что изменилось в витрине за прошлый месяц?»).

## Команды

Запуск инкрементального обновления:
```bash
python -m app.cli update-rag
```

Режим предпросмотра (без внесения изменений):
```bash
python -m app.cli update-rag --dry-run
```
